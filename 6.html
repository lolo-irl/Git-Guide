Branches


Branches são ramificações do código em um repositório Git, permitindo que você trabalhe em diferentes versões de um projeto simultaneamente. Cada branch pode conter suas próprias alterações, independentemente das outras, o que é útil para desenvolver novas funcionalidades, corrigir bugs ou experimentar ideias sem afetar a versão principal do projeto, geralmente chamada de "main" ou "master".


Para criar uma nova branch no GitHub usando o Git no terminal, você pode seguir esses passos:



1. Primeiro, certifique-se de estar na branch a partir da qual deseja criar a nova branch. Use o comando 'git checkout main' (ou o nome da branch atual).


2. Crie a nova branch com o comando git checkout -b nome-da-branch, substituindo "nome-da-branch" pelo nome que você deseja dar à nova branch.


3. Depois de criar a branch, você pode começar a fazer commits nessa nova branch. Quando terminar, pode enviar suas alterações para o GitHub usando git push -u origin nome-da-branch.



Merges


Fazer um merge de branches significa combinar as mudanças de uma branch (geralmente uma branch de desenvolvimento ou feature) com outra, geralmente a branch principal (como main ou master). Isso é feito quando as alterações em uma branch estão prontas para serem integradas ao código principal do projeto.


Como fazer merge de branches:

1. Navegue para a branch principal: Primeiro, vá para a branch na qual você deseja fazer o merge. Normalmente, isso será a branch main ou master. Use o comando: bash git checkout main


2. Execute o merge: Em seguida, faça o merge da branch de desenvolvimento (por exemplo, feature-branch) na branch principal usando: bash git merge feature-branch. Isso aplicará as mudanças da feature-branch na main.

3. Enviar para o GitHub: Finalmente, envie as mudanças combinadas para o repositório remoto no GitHub usando: bash git push origin main
